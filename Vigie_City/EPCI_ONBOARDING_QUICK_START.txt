================================================================================
THE 5-STEP FLOW
================================================================================

Step 1: TERRITORY SELECTION
   → Choose "Commune" or "EPCI"
   → Search and select territory
   → Load related data (EPCIs or communes)

Step 2: ADMIN CONTACT
   → Enter email, name, password (for EPCI or commune admin)
   → Real-time validation:
     * Email: RFC 5322 format
     * Password: 8+ chars, 1 uppercase, 1 digit, 1 special char
   → Shows validation checklist

Step 3: CONFIGURATION
   Commune path:
     → Select pricing plan (Hameau/Village/Bourg/Métropole)
   EPCI path:
     → Table of communes in EPCI
     → Add admin email/name/phone for each
     → Add/remove rows dynamically

Step 4: PAYMENT DETAILS
   → Select payment date (default: today)
   → Choose payment type (Chorus Pro / Virement / Devis en attente)
   → Check validation checkbox
   → Displays estimated amount

Step 5: CONFIRMATION
   Before submit:
     → Shows summary of all entered data
     → "Valider & créer" button
   After submit:
     → Results display:
        * Communes created (count)
        * Communes failed (if any, with errors)
        * Created license IDs
        * Created admin user IDs
     → "Voir les communes" button to dashboard

KEY FEATURES
================================================================================

✅ DUAL-PATH SUPPORT
   - Single commune: Create one admin + license
   - EPCI: Create EPCI admin + multiple commune admins (batch)

✅ COMPREHENSIVE VALIDATION
   - Email: RFC 5322 format check
   - Password: Strength requirements with checklist
   - INSEE code: 5-digit format
   - Phone: French format support
   - Duplicates: Table deduplication in EPCI path

✅ STATE PRESERVATION
   - Formdata object preserved across steps
   - Back navigation preserves entered data
   - No global state needed (pure React useState)

✅ ERROR HANDLING

   - Best-effort batch (continues despite individual failures)
   - Detailed error messages for each failed commune
   - Toast notifications for user feedback

✅ DATA FLOW
   - Form → Edge Function → DB
   - Payment fields populated automatically
   - User roles created (admin_epci or admin)
   - Licenses created with expiry dates
   - Welcome emails sent to admins
