FUNCTION 1: freemium-auto-filter
File: supabase/functions/freemium-auto-filter/index.ts
Size: 3,262 bytes
Purpose: Auto-filter incoming reports for spam/profanity

Algorithm:
- Detects spam patterns (Viagra, casino, limited offers, URLs) → 0.2 per pattern
- Detects ALL CAPS aggression (10+ chars) → 0.15
- Detects excessive punctuation (!!! or ???) → 0.1
- Detects French profanity (putain, connard, etc.) → 0.25
- Score capped at 1.0

Behavior:
- If score > 0.7: sets visible_to_public=false, status=pending_review
- If score ≤ 0.7: sets visible_to_public=true, status=public
- Creates moderation_queue entry if flagged

Input: POST with { report_id, content, municipality_id }
Output: { success, report_id, auto_filter_score, visible_to_public, status }

FUNCTION 2: city-fetch-reports
File: supabase/functions/city-fetch-reports/index.ts
Size: 3,290 bytes
Purpose: Fetch reports for city admin dashboard during Phase 2

Features:
- Requires Bearer token in Authorization header
- Verifies user is admin of an active subscription municipality
- Returns all reports for that municipality, categorized:
  * pending_response: status='pending_review'
  * resolved: status='public' with city_response set
  * escalated: status='escalated'

Input: POST with Authorization header
Output: {
  municipality_count,
  municipalities: [{ id, name }],
  reports: {
    total,
    pending_response,
    resolved,
    escalated,
    data: { pending[], resolved[], escalated[] }
  }
}
